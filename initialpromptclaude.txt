## 2. Core Business Logic Engine (Austrian Rules 2025/2026)
Create an isolated, highly testable utility module `AustrianTaxCalculator.ts`. It must strictly adhere to these rules:

**A. Taggeld (Per Diem)**
- **Base Rate:** €30.00 per 24 hours (Domestic).
- **The 3-Hour Trigger:** Trip duration MUST be strictly > 3.0 hours. If <= 3.0, Taggeld = €0.
- **Aliquot Calculation:** Calculated per started hour as 1/12th of the daily rate.
  - *Formula:* `Math.ceil(tripDurationInHours) * (30.00 / 12)`
  - *Cap:* Max €30.00 per calendar day.
- **Meal Deductions (Kürzung):** If meals are employer-paid:
  - 1 Meal = Deduct €15.00 from the calculated Taggeld.
  - 2+ Meals = Taggeld drops to exactly €0.00.
  - *Edge Case:* The deduction can NEVER result in a negative Taggeld. Floor at €0.00.

**B. Kilometergeld (Mileage)**
- **Base Rate:** €0.50 per km (Car).
- **Annual Cap Limit:** 30,000 km per User per calendar year. Any km claimed where `ytdMileage + currentTripMileage > 30000` must yield €0 tax-free payout for the excess.

**C. The 5/15 Day Rule (Mittelpunkt der Tätigkeit)**
- If a User travels to the EXACT same destination for > 5 consecutive days OR > 15 days in a calendar year, that destination becomes a secondary workplace. Tax-free Taggeld for this destination drops to €0.

**D. Kollektivvertrag (KV) Splitter**
- If a User's KV allows a higher daily rate (e.g., €35), the engine must output a split: `taxFreeAmount`: €30.00, `taxableAmount`: €5.00.

## 3. Database Schema Requirements (Supabase/Prisma)
- `User`: id, name, email, role (USER/ADMIN), kollektivvertragRate (Float), ytdMileage (Int).
- `Trip`: id, userId, destination (String), startTime (DateTime), endTime (DateTime), status (DRAFT/PENDING/APPROVED/REJECTED).
- `ExpenseLine`: id, tripId, type (MILEAGE, MEAL_DEDUCTION, RECEIPT), amount (Float), vatRate (Enum: 10, 13, 20), description.
- `Receipt`: id, expenseLineId, storagePath (Supabase Storage URL), originalAmount.

## 4. UI/UX Requirements
- **Dashboard:** Sleek overview showing YTD Mileage progress, total payouts, and pending trips. Use a German UI (e.g., "Neue Reise", "Belege").
- **Trip Creation Flow:** Use Shadcn's DatePicker (with time selection) to ensure precision. Provide real-time calculation previews of the Taggeld based on hours and checked meals.
- **Receipt Upload:** Clean drag-and-drop zone uploading to Supabase Storage, then passing through `tesseract.js` to extract amounts/dates.
- **Admin Approval View:** A data table for HR to batch-approve trips. Approved trips become immutable (read-only) for BAO audit compliance.
- **BMD Export:** A server action to download approved trips as a CSV (`Date, Account, Amount, VAT_Code, Cost_Center, Description`).
